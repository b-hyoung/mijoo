# 나스닥 대형주 주가 예측 시스템 — 설계 문서

**작성일:** 2026-04-07  
**스택:** Next.js · FastAPI · XGBoost · Docker Compose

---

## 개요

나스닥100 대형주의 **2주 / 4주** 후 가격을 예측하는 시스템.  
기술적 지표 + 뉴스 감성 + 공매도 데이터 + 매수/매도 주문 흐름을 결합해 예측하고, 웹 대시보드와 Discord 알림으로 결과를 전달한다.

> 1주 예측은 제거. 대신 **최근 1개월 매수/매도 누적 흐름**을 수집·시각화하고, 최근 1주 흐름을 2주/4주 예측의 핵심 피처로 활용한다.

---

## 시스템 구성 (역할 분리)

### 1. Data Collector (수집기)
**역할:** 외부 소스에서 원시 데이터를 수집하고 저장

| 소스 | 데이터 | 방식 |
|------|--------|------|
| yfinance | OHLCV 주가 히스토리 | Python 라이브러리 |
| NewsAPI / Yahoo RSS | 종목별 최신 뉴스 기사 | HTTP 요청 |
| Finviz | Short Float % (공매도 비율) | HTML 스크래핑 |
| Finra | 공식 공매도 데이터 (2주 딜레이) | CSV 다운로드 |

- 매일 장 마감 후 자동 실행 (APScheduler)
- 나스닥100 전체 + 사용자 커스텀 종목 지원
- 수집된 데이터는 로컬 DB(SQLite)에 저장

---

### 2. Feature Engineer (피처 생성기)
**역할:** 원시 데이터 → ML 모델 입력 피처 변환

**기술적 지표 피처:**
- 이동평균선: MA5, MA20, MA60
- RSI (14일)
- MACD + Signal Line
- 볼린저 밴드 (상단/하단 이격)
- 거래량 변화율 (5일 평균 대비)

**뉴스 감성 피처:**
- 최근 7일 기사 수집
- OpenAI API로 기사별 bullish / bearish / neutral 점수화
- 종목별 감성 평균 점수 (−1.0 ~ +1.0)

**공매도 피처:**
- Short Float % (Finviz)
- 공매도 증감 추이 (전주 대비 변화율)

**매수/매도 흐름 피처 (Order Flow):**
- 최근 1개월 일별 매수/매도 거래량 수집
- OBV (On-Balance Volume) 추세
- 거래량 대비 가격 변화율 (가격 정체 + 거래량 급증 → 매집 시그널)
- 업틱/다운틱 비율
- **최근 1주 누적 흐름 → 2주/4주 예측 피처로 직접 투입**

---

### 3. AI Debate Engine (다중 페르소나 분석)
**역할:** 강세/약세 논거 생성 → 조율 → 최종 판정

#### 페르소나 구성 (6개 + Orchestrator + Judge)

| 페르소나 | 포지션 | 담당 데이터 |
|----------|--------|-------------|
| 뉴스 애널리스트 (강세) | Bullish | 긍정적 뉴스 논거 |
| 뉴스 애널리스트 (약세) | Bearish | 부정적 뉴스 논거 |
| 기술적 분석가 (강세) | Bullish | 상승 지표 해석 |
| 기술적 분석가 (약세) | Bearish | 하락 지표 해석 |
| 공매도 전문가 (강세) | Bullish | 숏스퀴즈 가능성 |
| 공매도 전문가 (약세) | Bearish | 공매도 압력 분석 |
| **Orchestrator** | 중립 | 비율 조율 + 재호출 결정 |
| **Judge** | 중립 | 최종 판정 + 요약 생성 |

#### 실행 흐름

```
1. 6개 페르소나 병렬 호출 (Claude API)
        ↓
2. Orchestrator 체크
   - 강세/약세 논거 균형 확인
   - 한쪽 압도 시 반대측 재호출
   - 시장 상황에 따라 가중치 조정
     · 어닝시즌 → 뉴스 비중 ↑
     · 공매도 급변 → 공매도 전문가 비중 ↑
     · 기술적 변곡점 → 기술 분석가 비중 ↑
   - 필요 시 추가 라운드 (최대 3라운드)
        ↓
3. Judge AI — 최종 판정
   - 방향: 상승 / 하락
   - 신뢰도: % (논쟁 팽팽 → 50%대, 한쪽 압도 → 80%+)
   - 요약: 1~2줄
```

### 4. ML Engine (예측 엔진)
**역할:** 피처 → 가격 예측

- 모델: XGBoost (종목별 × 2개 — 2주 / 4주)
- 학습 데이터: 과거 5년치
- 출력:
  - **방향:** 상승 / 하락 + 신뢰도 (%) — AI Debate Engine과 교차 검증
  - **예상 가격 범위:** 하단 ~ 상단
- 모델 파일 로컬 저장, 주 1회 재학습

---

### 4. Backend API (FastAPI)
**역할:** 예측 결과 제공 + 알림 발송

**엔드포인트:**
```
GET /predict/{ticker}         → 4주치 예측 결과
GET /stocks/list              → 추적 중인 종목 목록
GET /history/{ticker}         → 과거 가격 데이터
POST /stocks/custom           → 커스텀 종목 추가
DELETE /stocks/custom/{ticker}→ 커스텀 종목 제거
GET /settings                 → 알림 설정 조회
PUT /settings                 → 알림 임계값 변경
```

**Alert Engine:**
- 예측 변동이 설정한 임계값(%) 이상일 때 Discord 즉시 알림
- 매주 월요일 09:00 주요 종목 요약 리포트 Discord 발송

---

### 5. Discord Notifier (알림)
**역할:** 조건 충족 시 Discord 웹훅으로 메시지 발송

**즉시 알림 포맷:**
```
⚡ NVDA 급등 예측
• 2주 후 방향: 📈 상승 (신뢰도 76%)
• 예상 가격: $891 ~ $924
• 근거: 실적 호조 기사, 공매도 감소, RSI 회복
```

**주간 리포트 포맷:**
```
📊 주간 예측 요약 (2026-04-07)
• AAPL: 📈 +4.2% 예상
• NVDA: 📈 +7.1% 예상
• TSLA: 📉 −2.8% 예상
```

---

### 6. Frontend Dashboard (Next.js)
**역할:** 예측 결과 시각화

**페이지 구성:**
- `/` — 종목 카드 목록 (방향 + 예상가 + 근거 요약)
- `/stock/[ticker]` — 개별 종목 상세
  - 과거 가격 차트 + 2주/4주 예측 오버레이
  - **1개월 매수/매도 누적 흐름 차트** (OBV + 거래량 분포)
- `/settings` — 커스텀 종목 추가/제거, 알림 임계값 설정

**카드 UI 예시:**
```
NVDA
📈 상승 76%  |  2주 후 $891~$924
근거: 실적 호조 · 공매도 감소 · 1주 매집 시그널 감지
```

**1개월 흐름 차트:**
- 일별 매수/매도 거래량 바 차트 (초록/빨강)
- OBV 라인 오버레이
- "조용한 매집" 구간 자동 하이라이트

---

### 7. Infra (Docker Compose)
**역할:** 전체 시스템 로컬 실행

```yaml
services:
  frontend:   # Next.js — localhost:3000
  backend:    # FastAPI  — localhost:8000
  scheduler:  # 수집 + 재학습 자동화
```

```bash
docker-compose up   # 한 방에 전체 실행
```
